# SightEdit システム要件

## 概要
SightEditはChrome拡張機能として動作するVisual Markdownエディターです。Word形式エクスポート機能を含む高機能を提供するための推奨システム要件を以下に示します。

## 必要要件

### ブラウザ要件
- **Google Chrome**: バージョン 88 以上 (推奨: 最新版)
- **Chromium系ブラウザ**: Microsoft Edge, Brave, Opera等 (Chrome 88互換)

### オペレーティングシステム
- **Windows**: Windows 10 以上 (推奨: Windows 11)
- **macOS**: macOS 10.15 Catalina 以上 (推奨: macOS 12 Monterey以上)
- **Linux**: Ubuntu 18.04 LTS 以上、または同等のディストリビューション

### ハードウェア要件

#### 最小要件
- **メモリ (RAM)**: 4GB
- **CPU**: デュアルコア 2.0GHz
- **ストレージ**: 500MB の空き容量
- **ネットワーク**: インターネット接続（AI機能使用時）

#### 推奨要件
- **メモリ (RAM)**: 8GB 以上
- **CPU**: クアッドコア 2.5GHz 以上
- **ストレージ**: 2GB の空き容量
- **ネットワーク**: ブロードバンドインターネット接続

#### 高性能要件 (大容量ドキュメント対応)
- **メモリ (RAM)**: 16GB 以上
- **CPU**: 8コア 3.0GHz 以上 (Intel i7, AMD Ryzen 7相当)
- **ストレージ**: SSD推奨、4GB の空き容量

## 機能別要件

### 基本エディター機能
- **メモリ**: 1GB～2GB
- **ドキュメントサイズ**: 〜10KB (約5,000文字)
- **対応要素**: テキスト、見出し、リスト、基本的な書式

### Word エクスポート機能

#### 軽量版変換 (10KB～50KB ドキュメント)
- **メモリ**: 2GB～4GB
- **処理時間**: 数秒～10秒
- **対応要素**: 見出し、段落、太字、表（基本）
- **制限**: 複雑な表、画像は簡易変換

#### 標準版変換 (〜10KB ドキュメント)
- **メモリ**: 4GB～8GB
- **処理時間**: 1秒～5秒
- **対応要素**: 全Markdown要素（表、画像、コードブロック等）
- **品質**: 高品質なレイアウト保持

#### チャンク処理 (50KB以上 大容量ドキュメント)
- **メモリ**: 8GB以上推奨
- **処理時間**: 10秒～30秒
- **対応要素**: テキスト、見出し、表（テキスト形式）
- **制限**: 軽量処理によりフォーマットは簡素化

### AI機能
- **メモリ**: 4GB以上
- **ネットワーク**: 常時接続必須
- **API要件**: Gemini APIキー または Claude アクセス

## パフォーマンス指標

### ドキュメントサイズ別性能

| ドキュメントサイズ | 変換方式 | 必要メモリ | 処理時間 | 表対応 |
|----------------|---------|-----------|---------|--------|
| ～10KB | 標準変換 | 4GB | 1-5秒 | 完全対応 |
| 10KB～50KB | 軽量変換 | 2-4GB | 5-10秒 | 基本対応 |
| 50KB～200KB | チャンク処理 | 8GB+ | 10-30秒 | テキスト形式 |
| 200KB以上 | 分割推奨 | - | - | - |

### メモリ使用量の目安
- **アイドル状態**: 100MB～200MB
- **編集中**: 500MB～1GB
- **Word変換中**: 2GB～8GB (ドキュメントサイズによる)
- **AI処理中**: +500MB～1GB

## トラブルシューティング

### メモリ不足エラーの対処法

1. **ドキュメント分割**: 大きなドキュメントを複数の小さなファイルに分割
2. **他タブを閉じる**: Chromeの他のタブやアプリケーションを終了
3. **ブラウザ再起動**: Chromeを再起動してメモリをクリア
4. **軽量モード**: 設定で軽量変換モードを有効化

### 処理速度が遅い場合

1. **CPU使用率確認**: タスクマネージャーで他のプロセスを確認
2. **バックグラウンドアプリ終了**: 不要なアプリケーションを終了
3. **ディスク容量確認**: 十分な空き容量があることを確認
4. **拡張機能無効化**: 他のChrome拡張機能を一時的に無効化

### 変換エラーの場合

1. **フォールバック確認**: コンソールでフォールバック処理を確認
2. **ドキュメント検証**: 特殊文字や壊れた表記法がないかチェック
3. **段階的テスト**: 小さなドキュメントから徐々に容量を増やしてテスト

## 最適化のヒント

### パフォーマンス向上
- **SSDの使用**: 処理速度向上のためSSD推奨
- **十分なRAM**: メモリ不足を避けるため推奨要件以上を確保
- **定期的なブラウザクリーンアップ**: キャッシュとデータのクリア

### 効率的な使用法
- **適切なドキュメントサイズ**: 1ファイルあたり10KB以下を目安
- **段階的な保存**: 大きな変更の前に作業内容を保存
- **プレビュー活用**: Word変換前にプレビューで確認

## 互換性情報

### 対応ファイル形式
- **入力**: Markdown (.md), プレーンテキスト (.txt)
- **出力**: DOCX, PDF, HTML, Markdown
- **インポート**: DOCX（基本的なフォーマットのみ）

### ブラウザ互換性
- **推奨**: Google Chrome 最新版
- **対応**: Chromium系ブラウザ（Edge, Brave, Opera等）
- **非対応**: Firefox, Safari

## サポート情報

### ログ出力
パフォーマンス問題の診断には、ブラウザのデベロッパーツール（F12）のConsoleタブでログを確認してください。

### 報告用情報
問題報告時は以下の情報を含めてください：
- OS・ブラウザバージョン
- ドキュメントサイズ
- メモリ使用量
- エラーメッセージ
- コンソールログ
